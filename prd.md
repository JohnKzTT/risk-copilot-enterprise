# PRD — Risk Co-Pilot Enterprise
**Product Requirements Document**
Version 1.0 | Дата: июнь 2025 | Статус: Draft

---

## 1. Обзор продукта

**Risk Co-Pilot Enterprise** — внутренняя веб-платформа управления рисками для банка, объединяющая количественную аналитику, регуляторную отчётность и AI-ассистирование в едином интерфейсе.

### Проблема
Риск-менеджеры банка работают в разрозненных системах: Excel для VaR/стресс-тестов, Word для писем в АРРФР, отдельные базы ВНД, внешние консультации по МСФО 9 и Базелю. Это замедляет принятие решений, увеличивает операционный риск и снижает качество регуляторной отчётности.

### Решение
Единая платформа с 8 функциональными модулями, где количественные данные, регуляторные документы и AI-генерация объединены в одном рабочем пространстве.

### Целевые пользователи
| Роль | Основные сценарии |
|------|-------------------|
| Chief Risk Officer | KPI-обзор, записки на Правление, stress-сценарии |
| Риск-аналитик (рыночный риск) | VaR dashboard, CoR матрица, backtesting |
| Риск-аналитик (кредитный риск) | МСФО 9 ECL, кредитные KPI, ВНД |
| Compliance / Регуляторный отдел | Письма в АРРФР, Базель III раскрытия |
| Финансовый контролёр | KPI-анализ отклонений, МСФО 9 |

## 2. Цели и метрики успеха

### Бизнес-цели
- Сократить время подготовки письма в АРРФР с 4 часов до 20 минут
- Снизить время подготовки материалов на Правление с 1 дня до 2 часов
- Централизовать риск-отчётность: 0 разрозненных Excel-файлов для VaR/стресс-тестов

### KPI платформы
| Метрика | Цель |
|---------|------|
| Time-to-report (АРРФР письмо) | ≤ 20 мин |
| Time-to-report (Записка на Правление) | ≤ 2 ч |
| Охват ВНД в поиске | 100% действующих документов |
| NPS риск-менеджеров | ≥ 70 |
| Ежедневные активные пользователи | ≥ 15 (в пределах риск-блока) |

## 3. Функциональные требования

### 3.1 CoR Dashboard — Матрица корреляций

**Описание:** Визуализация корреляционных зависимостей между классами активов.

**Функции:**
- Интерактивная тепловая карта 5×5 (FX / Rates / Equity / Commodity / Credit Spread)
- Цветовая кодировка: зелёный (<0.3) → жёлтый (0.3–0.5) → оранжевый (0.5–0.7) → красный (≥0.7)
- Карточки парных корреляций с прогресс-баром
- График динамики скользящей корреляции (окно 60 дней)
- Коэффициент корреляции Пирсона по историческим данным

**Источник данных:** Рыночные данные (Bloomberg / внутренний MDM)
**Обновление:** real-time (внутри торговой сессии)

---

### 3.2 VaR Dashboard — Расчёт и бэктестинг Value-at-Risk

**Описание:** Центральный экран управления рыночным риском с бэктестингом по Базель III.

**Функции:**
- KPI-карточки: VaR 99% (1д), VaR 95% (1д), Stressed VaR, число превышений (Купер-тест)
- Area-chart: VaR 99% / VaR 95% / фактический P&L с выделением нарушений красными точками
- Зона Базель III: автоматическое определение цвета зоны (зелёная ≤4, жёлтая 5–9, красная ≥10 нарушений)
- Декомпозиция VaR по деску (FX / Rates / Equity / Credit / Commodity)
- Сравнительный bar-chart Stressed VaR по историческим кризисным сценариям
- Reference line на нулевой отметке P&L

**Методология:** Историческая симуляция, 1-day holding period, 250-дневное окно
**Соответствие:** Базель III / BCBS 352 (бэктестинг), ФРТБ (Stressed VaR)

---

### 3.3 Stress Testing — Стресс-тестирование портфеля

**Описание:** Сценарный анализ воздействия кризисных условий на баланс банка.

**Функции:**
- 6 предустановленных сценариев: Базовый, Негативный, Кризис 2008, COVID-19 March'20, Шок регулятора, Геополитический
- Переключение сценариев с мгновенным обновлением KPI-карточек (потери / вероятность / остаточный капитал)
- Горизонтальный bar-chart сравнения потерь по всем сценариям
- Таблица влияния на CET1 ratio с проверкой регуляторного минимума 8.5%
- Выделение активного сценария во всех визуализациях
- Визуальная индикация нарушения минимального капитального буфера ($250M)

**Регуляторная база:** АРРФР Постановление №24 (стресс-тестирование), Базель III Pillar 2

### 3.4 Генерация писем в АРРФР

**Описание:** AI-генерация официальной корреспонденции с регулятором.

**Шаблоны (6 штук):**
1. Нарушение лимита ВаР
2. Ежемесячный отчёт по LCR
3. Результаты стресс-тестирования
4. Раскрытие по Базель III (Pillar 3)
5. Уведомление о росте NPL
6. Произвольный запрос

**Функции:**
- Выбор шаблона → поле для дополнительного контекста (данные, даты, причины)
- AI-генерация (Claude Sonnet) официального письма на русском языке
- Структура: шапка (кому/от кого/дата/исх. номер), основная часть, приложения, подпись
- Автоматические ссылки на нормативные акты РК (Постановления АРРФР, НБ РК)
- Кнопка «Копировать» для вставки в Word / корпоративную почту
- Реквизиты банка подставляются автоматически из конфигурации

**AI-промпт контекст:** роль — эксперт по банковскому регулированию РК, стиль — официальный деловой, язык — русский

---

### 3.5 Генерация записок на Правление

**Описание:** AI-генерация служебных меморандумов для Правления Банка.

**Типы записок (5 штук):**
1. Обзор рисков — квартальный
2. Изменение лимитов риска
3. Результаты стресс-тестирования
4. Риски нового продукта/операции
5. Риск-инцидент

**Функции:**
- Выбор типа + текстовое поле с данными/фактами
- AI-генерация меморандума с жёсткой структурой:
  - Заголовок (кому / от кого / дата / номер / тема)
  - Executive Summary (3–4 предложения)
  - Текущая ситуация с цифрами
  - Анализ рисков и ключевые выводы
  - Нумерованные рекомендации Правлению
  - Проект решения Правления
- Стиль: официальный, структурированный, конкретный (с цифрами)
- Копирование готового текста

**AI-промпт контекст:** роль — Chief Risk Officer казахстанского банка

---

### 3.6 AI-анализ отклонений KPI

**Описание:** Интерактивный выбор KPI с AI-диагностикой причин отклонений.

**Отслеживаемые KPI (8 показателей):**
| KPI | Плановое значение | Единица |
|-----|------------------|---------|
| VaR (99%) | 6.0 | млн $ |
| LCR | 115 | % |
| NSFR | 110 | % |
| CET1 | 10.5 | % |
| NPL | 4.0 | % |
| ROE | 15.0 | % |
| Cost/Income | 55 | % |
| Кредитная экспозиция | 3 000 | млн $ |

**Функции:**
- Карточки KPI с цветовой индикацией статуса: ok (зелёный) / warn (жёлтый) / crit (красный)
- Мультиселект KPI для анализа
- Grouped bar-chart план vs факт
- AI-анализ по каждому выбранному KPI:
  - 2–3 возможные причины отклонения
  - Оценка рисков для банка
  - Конкретные рекомендации по устранению
  - Регуляторные последствия (АРРФР / Базель III)
- Итоговая интегральная оценка + топ-3 приоритетных действия

---

### 3.7 Поиск по ВНД Банка

**Описание:** Централизованный реестр внутренних нормативных документов с поиском и AI-ассистентом.

**Метаданные документа:**
- Номер (ID), полное название, дата редакции, версия, подразделение-владелец

**Типы документов:** Политики (П), Методики (МЕ), Инструкции (ИН), Положения (ПЛ)

**Функции поиска:**
- Полнотекстовый поиск по названию, номеру документа, подразделению
- Real-time фильтрация при наборе
- Список результатов с выделением совпадений
- Кнопка перехода к документу (↗)

**AI-ассистент по ВНД:**
- Режим включается отдельным тоглом
- Вопросы на естественном языке: «Какой лимит ОВП по ПЛ-003?», «Как рассчитывается VaR согласно МЕ-014?»
- AI отвечает со ссылками на конкретные номера и версии документов
- Контекст: полный список ВНД + нормы законодательства Казахстана

---

### 3.8 AI-чат по МСФО 9 и Базелю

**Описание:** Специализированный AI-ассистент по методологии финансовых стандартов.

**Область знаний:**
- МСФО 9: классификация (SPPI-тест, бизнес-модель), ECL (Stage 1/2/3), PD, LGD, EAD, SICR, ставка дисконтирования, форвардные макро-факторы
- Базель III: CET1, Tier 1/2, RWA, LCR, NSFR, леверидж, IRRBB, Pillar 2
- ФРТБ: IMA vs SA, RRAO, SES, DRC, ES vs VaR
- Операционный риск: AMA, BIA, TSA
- Казахстанский контекст: нормативы АРРФР, НБ РК

**Функции:**
- Полноценный диалоговый чат с историей сессии
- 6 быстрых вопросов-подсказок для старта:
  - «Как рассчитывается ECL для Stage 2 по МСФО 9?»
  - «Что такое SICR и критерии перевода в Stage 2?»
  - «Чем отличается LCR от NSFR в Базель III?»
  - «Как ФРТБ меняет расчёт рыночного риска?»
  - «Формула расчёта CET1 ratio»
  - «PD × LGD × EAD — как применять в ECL?»
- Анимированный typing indicator
- Ответы с формулами, конкретными цифрами, ссылками на параграфы стандартов

## 4. Нефункциональные требования

### Производительность
- Загрузка модуля: < 1 сек
- AI-генерация письма / записки: < 15 сек
- AI-чат ответ: < 10 сек
- Поиск по ВНД: real-time (< 50 мс)

### Безопасность
- Аутентификация: SSO через корпоративный IdP (LDAP / Active Directory)
- Авторизация: RBAC (CRO / Analyst / Viewer)
- Данные: шифрование в transit (TLS 1.3) и at rest (AES-256)
- AI API: запросы не содержат персональных данных клиентов банка
- Аудит-лог: все запросы к AI-модулям логируются с user ID и timestamp

### Доступность
- SLA: 99.5% uptime (исключая плановые работы)
- Браузеры: Chrome 110+, Edge 110+, Firefox 110+
- Разрешения: от 1280×768
